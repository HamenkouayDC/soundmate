ЗАЯВКА НА ДОСТУП К SOUNDCLOUD API
====================================

Куда подавать: https://developers.soundcloud.com -> Apply for API access

Готовые формулировки для полей формы (можно копировать как есть
или адаптировать под точные поля, которые покажет форма на сайте —
SoundCloud периодически их меняет):

---

ПОЛЕ: Название приложения / Application name
Soundmate

---

ПОЛЕ: Описание проекта / Project description

Soundmate — учебный проект (студенческая/стажёрская разработка),
приложение для социального знакomства, в котором пользователи находят
друг друга на основе схожести музыкальных предпочтений, а не анкетных
данных. Алгоритм матчинга анализирует аудио-характеристики прослушиваемой
музыки (энергичность, темп, настроение трека) и сопоставляет пользователей
с похожим музыкальным профилем.

Для построения музыкального профиля пользователя нам нужен доступ к данным
о треках и плейлистах, которые пользователь подключает через SoundCloud,
в дополнение к уже используемым Spotify API и Last.fm API. Использование
строго read-only: мы не публикуем контент, не модифицируем плейлисты
пользователей и не передаём полученные данные третьим лицам.

---

ПОЛЕ: Какие данные/эндпоинты планируете использовать
- Получение профиля пользователя и списка его треков/лайков (для построения
  музыкального вкуса)
- Получение метаданных треков (жанр, теги, длительность)
- OAuth для авторизации пользователей, подключающих свой SoundCloud-аккаунт

ПОСЛЕ ОТПРАВКИ:
SoundCloud рассматривает заявки не мгновенно — иногда это несколько дней
или недель, поэтому задача правильно стоит в плане первой недели: подать
сейчас, не блокируясь на ответе, и продолжать работу со Spotify/Last.fm,
пока ждём подтверждения.

Результат, который нужно зафиксировать в отчёте по спринту:
"Заявка на SoundCloud API подана [дата]. Ответа пока нет / получен доступ,
client_id: ..." — в зависимости от того, что произошло на момент отчёта.
