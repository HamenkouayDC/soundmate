# 🛠 Настройка GitHub для Soundmate — пошаговая инструкция

## Шаг 1 — Создать репозиторий на GitHub

1. Зайти на [github.com/new](https://github.com/new)
2. Заполнить:
   - **Repository name:** `soundmate`
   - **Description:** `Сервис знакомств по музыкальному вкусу — «Твоя музыка найдёт тебя»`
   - **Visibility:** Private (учебный проект, потом можно сделать публичным)
   - ❌ НЕ ставить галочку «Add README» — мы добавим свой
3. Нажать **Create repository**

---

## Шаг 2 — Залить структуру репозитория

Скачай архив с файлами (прилагается) и выполни в терминале:

```bash
# Разархивируй и перейди в папку
cd soundmate

# Инициализируй git
git init

# Добавь remote (замени ВАШ_ЛОГИН)
git remote add origin https://github.com/ВАШ_ЛОГИН/soundmate.git

# Первый коммит
git add .
git commit -m "init: project structure and README"
git branch -M main
git push -u origin main
```

---

## Шаг 3 — Добавить участников команды

1. Открыть репозиторий → **Settings** → **Collaborators**
2. Нажать **Add people**
3. Добавить по GitHub-логину каждого участника:
   - Backend-разработчик
   - Frontend-разработчик
   - UX/UI дизайнер
   - ML-инженер
   - Бизнес-аналитик
4. Выбрать роль **Write** для всех (позволяет пушить ветки и создавать PR)

---

## Шаг 4 — Создать GitHub Project (доска задач)

1. Открыть вкладку **Projects** в репозитории
2. Нажать **Link a project** → **New project**
3. Выбрать шаблон **Board** (канбан с колонками)
4. Назвать: `Soundmate Sprint 1`

### Настроить колонки
Переименовать колонки в:
- **📋 To Do** — задачи недели, ещё не начатые
- **🔄 In Progress** — в работе прямо сейчас
- **👀 Review** — сделано, ждёт проверки
- **✅ Done** — завершено

### Добавить поля к карточкам
Нажать **+** в правом верхнем углу таблицы → **New field**:
- `Роль` — тип **Single select**: Тимлид, Backend, Frontend, UX/UI, ML, Бизнес-аналитик
- `Приоритет` — тип **Single select**: 🔴 Высокий, 🟡 Средний
- `Неделя` — тип **Single select**: Неделя 1, Неделя 2

---

## Шаг 5 — Завести карточки задач

Создать Issues для каждой задачи из бэклога. Шаблон Issue:

**Название:** `[Роль] Название задачи`

Пример: `[ML] Протестировать Spotify API и Last.fm API`

**Тело:**
```
## Задача
Краткое описание

## Шаги выполнения
- [ ] Шаг 1
- [ ] Шаг 2
- [ ] Шаг 3

## Результат
Что должно быть по итогу
```

После создания Issue — добавить его в Project и выставить поля (Роль, Приоритет, Неделя).

> 💡 **Совет:** Issues можно создавать прямо из доски Project — нажать **+** в колонке To Do.

---

## Шаг 6 — Настроить ветки

Рекомендуемое соглашение по именованию веток:

```
feature/название-задачи     # новая фича
fix/описание-бага            # исправление
docs/что-добавлено           # документация
```

Примеры:
```
feature/auth-jwt
feature/spotify-api-client
docs/requirements-mvp
```

**Защита ветки main:**
1. **Settings** → **Branches** → **Add rule**
2. Branch name pattern: `main`
3. Включить: **Require a pull request before merging**
4. Это значит: никто не пушит напрямую в main, только через PR

## Частые вопросы

**Q: Как участнику клонировать репозиторий?**
```bash
git clone https://github.com/ВАШ_ЛОГИН/soundmate.git
cd soundmate
```

**Q: Как создать свою ветку и запушить изменения?**
```bash
git checkout -b feature/моя-задача
# ... делаем работу ...
git add .
git commit -m "feat: описание что сделано"
git push origin feature/моя-задача
# Затем открыть PR на GitHub
```

**Q: Что делать с .env?**
Скопировать `.env.example` в `.env` и заполнить своими ключами. Файл `.env` не коммитится (он в .gitignore).
```bash
cp .env.example .env
```
